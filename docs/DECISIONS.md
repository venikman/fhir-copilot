# DECISIONS

| ID | Date | Status | Decision | Why | Linked study |
|---|---|---|---|---|---|
| D-0001 | 2026-03-17 | provisional | Start with the existing six-agent topology plus deterministic router fallback. | It preserves the current `fhir-agents` mental model while the workflow-first cutover is still being evaluated. | `docs/tradeoffs/01-agent-partitioning.md` |
| D-0002 | 2026-03-17 | accepted | Externalize runtime agent profiles and routing hints into files. | Prompt/config diffs stay reviewable and can be tested independently of orchestration code. | `docs/tradeoffs/02-router-strategy.md` |
| D-0003 | 2026-03-17 | accepted | Ship a stub FHIR backend for first boot and defer Firely integration to the next slice. | This keeps the starter runnable without external infrastructure while preserving the tool contracts. | `docs/ARCHITECTURE.md` |
| D-0004 | 2026-03-17 | superseded by D-0007 | Wire direct OpenAI provider first for Agent Framework startup. | Official docs show direct OpenAI support with `OpenAIClient`, `AsAIAgent`, function tools, sessions, and streaming; it is the smallest first integration surface. | `docs/ARCHITECTURE.md` |
| D-0005 | 2026-03-17 | accepted | Use `IAgentRunner` interface with DI-time runner selection instead of runtime if/else. | Keeps each provider's wiring isolated behind a single class. Adding a new provider means one new class + one DI registration, no changes to `CopilotService`. | `src/FhirCopilot.Api/Services/IAgentRunner.cs` |
| D-0006 | 2026-03-17 | accepted | Add `FhirCopilot.ServiceDefaults` for OTEL, health checks, and HTTP resilience. | Aspire ServiceDefaults pattern gives structured logging, `/health` + `/alive`, and standard resilience handlers for free. OTEL export activates when `OTEL_EXPORTER_OTLP_ENDPOINT` is set. | `src/FhirCopilot.ServiceDefaults/Extensions.cs` |
| D-0007 | 2026-03-18 | accepted | Replace OpenAI compatibility layer with Google `GenerativeAI` native SDK for Gemini. | The native SDK avoids the OpenAI-to-Gemini translation layer, supports Gemini-specific features, and is maintained by Google. | commit `3292d7a` |
| D-0008 | 2026-03-18 | accepted | Implement automatic model fallback chain on HTTP 429 (rate limit). | Gemini preview models have aggressive rate limits. A configurable fallback chain (`GeminiModels` array) lets the runner try the next model without user intervention. Each model gets one attempt per request. | commit `229d355` |
| D-0009 | 2026-03-18 | accepted | Return `"unverified"` confidence on all LLM responses instead of deriving from citation count. | Confidence based on "did the LLM mention a FHIR reference" was actively misleading — correct answers without slash-delimited references were tagged "low". Honest labeling over false precision. | `docs/PROD_READINESS.md` |
| D-0010 | 2026-03-18 | accepted | Add structured error responses (502/504/400/500) on `/api/copilot` instead of raw exceptions. | Clients need parseable error codes, not stack traces. The `CopilotError(code, message)` envelope maps to HTTP status codes at the endpoint level. | commit `d3c1dd9` |
| D-0011 | 2026-03-18 | accepted | Add Aspire AppHost with FHIR backend as a parameter resource. | The AppHost gives a dev dashboard (traces, metrics, logs) and injects `Provider__FhirBaseUrl` via Aspire parameter resources, keeping the API project unaware of Aspire. | commit `8b81b46` |
| D-0012 | 2026-03-18 | accepted | Add `OpenAiCompatibleAgentRunner` + SignalR hub for local LLM development. | Local inference via LM Studio (or any OpenAI-compatible server) enables offline development on M-series Macs without API keys. SignalR hub adds WebSocket support alongside existing HTTP/SSE endpoints. | commit `8d242b4` |
| D-0013 | 2026-03-18 | accepted | Scope the project to demo/prototype — skip auth, audit logging, session durability, CI/CD. | These are real prod concerns but this project stays at demo scope against synthetic FHIR data. The `IAgentRunner` and `IFhirBackend` interfaces are ready for production implementations later. | `docs/PROD_READINESS.md` |
| D-0014 | 2026-03-18 | accepted | Use `.env` file loader that walks up the directory tree from CWD. | A single `.env` at the repo root works regardless of whether you `dotnet run` from the project dir or the repo root. No need for separate `.env` files per project. | `Program.cs` (`LoadEnvFile` function) |
| D-0015 | 2026-03-18 | accepted | Tests use `StubAgentRunner` via `WebApplicationFactory` — never call a real LLM. | Test determinism is non-negotiable. The stub runner executes real FHIR queries against `SampleFhirBackend` and returns structured answers, so tests validate routing, response shape, streaming, and OTEL without network calls. | `tests/FhirCopilot.Api.Tests/CopilotFixture.cs` |
| D-0016 | 2026-03-19 | accepted | Remove HTTP REST/SSE endpoints — SignalR hub is sole copilot transport. | SignalR `SendQuery`/`StreamQuery` are a complete superset of the REST/SSE endpoints. Removing duplicates reduces surface area and eliminates `SseWriter`. | this commit |
